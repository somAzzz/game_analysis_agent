## Closed gaps

### Orchestration and stale-data safety

- Policy aliases map `money -> work`, `visa -> admin`, and
  `burnout -> slacker`.
- `--scenario` is a first-class `sim`, `play`, and `all` dimension.
- `all` executes simulation/analysis, catalog export, every validator, optional
  LLM QA, and quality gates in a defined order.
- Validator subprocess failures, timeouts, missing files, malformed JSON, and
  unchanged prerequisite artifacts fail instead of silently reusing old
  reports. Reuse is an explicit opt-in.
- Matrix simulations use unique absolute outputs, preventing parallel cells
  from racing over one Godot `user://` report.
- Each matrix `--out` owns its cell report directories, so completed before
  and after executions cannot overwrite one another. `compare-matrix` verifies
  matching config hashes, cells, parameters, commands, seeds, and manifests,
  then independently revalidates simulation/boundary/persona evidence before writing
  `matrix_compare_summary.json` and per-cell structured diffs.

### Executable quality evidence

- `config/gates.yaml` is evaluated by a strict, fail-closed gate engine.
  Unknown schemas and corrupt/missing required inputs do not pass.
- Gate evaluation covers critical anomalies, cell-level distributions,
  action-group behavior, route distance, outcomes, design metrics, and
  interactive-agent quality where those inputs apply.
- Monte Carlo reports do not fail merely because play-only metrics are absent;
  recorded play reports remain strict.
- `coverage-v2` measures difficulty/policy/scenario cells, state regimes and
  transitions, events, choices, actions, and data-quality problems. Catalogs
  provide denominators and untriggered IDs when available.
- Anomalies retain seed/week/action/event replay context.

### Agent and statistical evaluation

- `agent_eval.json` reports first-pass/final valid rate, repair and fallback
  rate, illegal actions, invalid event choices, anomalies per five weeks, risk
  acknowledgement, persona alignment, LLM errors/latency, and final ending.
- The player audit includes all retry calls instead of only the final attempt.
- Re-running a playtest clears the old trace rather than appending incompatible
  steps.
- The interactive probe exports the same
  `RiskEvaluator.get_top_risks` rows used by the game. Python validates the
  contract and source week before prompting the player; compatibility fallback
  is explicit and retains its reason instead of silently mixing risk sources.
- Ending distributions are grouped by policy, difficulty, and scenario and
  include sample size plus Wilson 95% confidence intervals.

### Contracts and reproducibility

- Contract fixtures always run locally; a marked smoke test can validate real
  artifacts from `GAME_PROJECT_PATH`.
- `trace-manifest-v2` adds file modification times and provenance: agent/game
  Git commit and dirty state, Python/platform/Godot information, exact runtime
  source SHA-256, and config/prompt hashes.
- Each matrix cell has stable parameters, config and source hashes, timestamps, status, and
  report path in `cell_manifest.json`; matrix-level status is summarized
  separately.

Orchestration checks:

```bash
# Strictly parse and enumerate all 140 cells without Godot/LLM execution.
uv run python tools/run_gameplay_agent.py matrix --dry-run --jobs 4

# Run the same fixed-seed matrix into isolated before/after output roots.
uv run python tools/run_gameplay_agent.py matrix \
  --out reports/matrix/before --jobs 4
# Apply the code change under test; keep config/matrix.yaml unchanged.
uv run python tools/run_gameplay_agent.py matrix \
  --out reports/matrix/after --jobs 4

# Fail closed unless both executions contain strictly pairable evidence.
uv run python tools/run_gameplay_agent.py compare-matrix \
  --before reports/matrix/before \
  --after reports/matrix/after \
  --out reports/compare/matrix

# Run all Godot validators with fresh route/demo prerequisites.
uv run python tools/run_gameplay_agent.py validate \
  --report-dir reports/validation/<run_id>

# Re-evaluate a recorded Agent trace without an external call.
uv run python tools/run_gameplay_agent.py eval \
  --report-dir reports/play/<run_id>
```

## Remaining boundaries

1. **The game has three open demo-gate findings.** The test system exposes
   these as a non-zero full-pipeline result; changing game balance or product
   thresholds is outside this Agent test-system repair.
2. **No local live-LLM result.** A real persona playthrough needs vLLM, SGLang,
   or a DeepSeek-compatible endpoint. `agent_eval` can evaluate recordings,
   but it does not manufacture live-model evidence.
3. **Private sibling access is explicit.** CI requires the
   `STUDY_IN_GERMANY_TOKEN` secret with read access. Without it, the real
   contract job must fail rather than silently skip.

The next integration work should therefore prioritize resolving the three game
demo-gate findings and retaining one live-LLM persona run. The earlier missing
validator, gate, coverage, risk-guidance,
contract, matrix comparison, frontend-test, and Agent-evaluation layers are
implemented.
