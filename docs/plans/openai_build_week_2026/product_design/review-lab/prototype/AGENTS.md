# Prototype Instructions

Run the local server yourself and open the preview in the browser available to this environment. Do not give the user server-start instructions when you can run it.

Before making substantial visual changes, use the Product Design plugin's `get-context` skill when the visual source is unclear or no longer matches the current goal. When the user gives durable prototype-specific design feedback, preferences, or decisions, record them in `AGENTS.md`.

When implementing from a selected generated mock, treat that image as the source of truth for layout, component anatomy, density, spacing, color, typography, visible content, and hierarchy.

## Durable Review Lab decisions

- Figma is no longer part of this review workflow.
- The visual source is `option-1-actual-replay-review-v2.png`, supported by the
  six-Persona motion storyboard and generated roster asset.
- Every character is a human strategy Persona: Newbie, Study, Money, Social,
  Visa, or Slacker. Do not add a Codex icon, robot, or generic mascot.
- Persona summaries distinguish frozen contract inputs from observed behavior;
  never present subjective intelligence or ability scores.

  from the verified `playthrough-v1` evidence bundle.

  navigation, the production Judge page, or the production Decision Graph.
- The Review Lab was approved on 2026-07-17. Production implementation may
  proceed; this removable lab remains the isolated visual and interaction
  reference and must not be coupled to product navigation at runtime.
