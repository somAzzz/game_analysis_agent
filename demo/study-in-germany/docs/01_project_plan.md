# 《留德模拟器》项目规划

## 定位

《留德模拟器》是一款 2D UI-heavy 回合制人生模拟游戏。玩家扮演一名准备赴德留学的中国学生，在每周有限行动槽里平衡学业、德语、金钱、签证、社交和心理状态。

拆分为“游戏模拟器项目”和“分析 Agent 系统”后，本项目同时承担模拟器职责：维护游戏规则和内容数据，提供可复现、可观测、可批量运行的 headless 实验环境。分析 Agent 系统只读取本项目输出的 JSONL、CSV 和验证报告，不直接依赖 UI 场景或内部实现细节。

## MVP 范围

- 时间跨度：20 周，第一个学期。
- 核心循环：选择行动 -> 结算状态 -> 触发事件 -> 推进周数。
- 内容规模：30 张行动卡、50 个事件、5 个 NPC、6 个 Demo 结局。
- 技术目标：Godot 4.7、GDScript、Resource 风格数据、JSON 存档。

## 开发路线

### 第 1 阶段：纯文字原型

目标：验证核心循环好不好玩。

做这些：

- 一个 Dashboard。
- 10 个属性。
- 20 张行动卡。
- 20 个事件。
- 周结算。
- 3 个结局。

不要先做美术、动画、复杂 UI。

### 第 2 阶段：第一学期 Vertical Slice

目标：做出可玩的 30-60 分钟版本。

加入：

- 5 个 NPC。
- 邮件系统。
- 考试系统。
- 行政任务链。
- 存档。
- 音效和简单立绘。

同时完善 Agent 协作能力：

- headless 批量跑局。
- seed 可复现。
- 结构化 weekly trace。
- 基础 bot policy。
- scenario 开局配置。
- 内容校验报告。

### 第 3 阶段：完整留学流程

目标：2-4 小时完整体验。

加入：

- 城市差异：Kassel / München / Berlin / Aachen / Frankfurt。
- 专业差异：Informatik / Maschinenbau / Elektrotechnik / BWL / Design。
- 职业路线：HiWi / Werkstudent / Praktikum / Thesis / Job。
- 多结局。
- 二周目继承。

### 第 4 阶段：可发行版本

目标：Steam Demo / itch.io Demo。

加入：

- 成就系统。
- 多语言：中文 / 英文 / 德文。
- 设置菜单。
- 云存档可选。
- Steam 页面素材。
- 试玩反馈收集。

## 验收标准

- 项目启动后进入标题页。
- 新游戏能进入行动规划界面。
- 每周至少 6 个可选行动。
- 第 20 周后能进入考试结算和 6 类结局之一。
- 存档读档能恢复周数、属性、flags、关系和历史。
