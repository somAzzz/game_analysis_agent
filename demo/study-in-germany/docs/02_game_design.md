# 游戏设计

## 核心属性

- money：金钱，允许为负，负数提高压力和风险。
- energy：精力，行动资源和考试发挥来源。
- stress：压力，过高会压低考试成绩并触发崩溃事件。
- loneliness：孤独，影响心理路线和部分社交事件。
- hunger：饥饿度，越高代表越饿。每周生活漂移会增加饥饿，超过阈值会拖累精力、压力和学业。
- academic_progress：学业进度，考试核心来源。
- exam_readiness：备考掌握度，代表 Übung、往年题、答疑和考纲理解。它和学业进度分开，防止玩家只刷平时进度却没有应试把握。
- language：德语能力，影响行政、邮件、社交和求职。
- social：社交能力，影响组队、WG、NPC 关系。
- visa_progress：行政熟练度/材料处理能力，不在仪表盘作为普通进度条展示，主要影响 Termin、邮件、材料补交等检定。
- blocked_account_balance：冻结账户余额。2026 年学生签证常见资金证明按 `11,904 EUR/年`、`992 EUR/月`建模，每 4 周释放一次到可支配现金。
- career_progress：职业进度，Demo 中用于提前铺设实习路线。
- gpa_score：国内大学成绩背景，开局按难度随机，影响 APS 综合结果和大学申请档位。
- aps_knowledge：APS 专业复习度，可通过复习大学专业课、整理材料和语言表达训练提升。
- aps_score：APS 综合评分，通过审核后生成。不同分数对应不同德国大学申请档位。
- current_week_work_hours：本周已记录合法打工小时数。学期内超过 20 小时会设置 `work_limit_exceeded` 并触发合规风险事件。
- annual_work_half_days：年度工作半天累计。按 4 小时折算 1 个半天，长线版本用于检查 280 个半天上限。
- failed_courses：挂科/需要补考数量。Demo 期末考试未通过会增加该值并设置 `needs_retake`。

学校注册和居留许可不是普通数值条，而是隐藏 flag：

- `school_registered`：未完成时不能正常上课，第一次上课会触发卡课事件，期末可能进入注册失败结局。
- `registration_delayed`：错过学校注册窗口后设置。玩家即使之后通过 TestDaF，也需要等下学期注册，承受额外生活费、学业拖后和精神压力。
- `visa_valid`：未完成时中后期触发居留风险链。拖延、黑工或错误处理可能设置 `deportation_warning`，进一步恶化会设置 `deportation_order` 并进入被迫离境结局。
- `work_limit_exceeded` / `work_law_violation`：合法打工时长越线和进一步违规。前者会触发补救事件，后者可进入打工违规风险结局。

## APS 与申请季

Demo 从出发前申请季开始，`week <= 0` 时只开放 APS/申请季行动。不同难度会随机生成初始资金、大学成绩、德语水平、专业复习基础、压力和社交基础。

APS 是显性门槛：

- 参加 APS 审核需要 `money >= 250`、`language >= 30`、`aps_knowledge >= 45`，并先完成 `aps_documents_ready`。
- 不达标时，玩家需要通过 APS 语言课、申请季打工、复习大学专业课、整理 APS 材料来补齐。
- APS 通过后生成 `aps_score`，并给出大学申请档位：精英/TU9 冲刺、研究型大学、综合大学/应用科学大学、保底项目/预科或受限专业谨慎。
- APS 通过后进入选校定位事件，然后推进到德国第一学期。

TestDaF 是独立语言门槛：

- `language` 表示德语学习/备考能力，但不能直接替代语言证书。
- TestDaF 显示为四项小分：阅读、听力、写作、口语。
- 德语授课项目入学要求按 `4x4` 建模：四项都达到 TDN 4 才设置 `testdaf_passed`。
- 出国前可以通过国内 TestDaF 备考和考试解决；如果没过，也可以抵达德国后继续读语言班和重考，但德国本地语言班、考试、住宿和生活费叠加，成本高很多。
- 学校注册 `school_registered` 需要 `testdaf_passed`，否则会触发语言证明卡注册和学业延迟事件。正常注册窗口到第 6 周结束；第 7 周仍未注册会触发错过窗口事件，后续只能走下学期注册路线。

## 行动卡

行动卡必须有取舍。申请季打工补 APS 费用但挤压复习，APS 语言课提升表达但花钱，复习大学知识提升面谈把握但增压。抵达德国后，打工补钱但耗精力，学习提成绩但增压，休息降压但拖进度，做饭和聚餐降低饥饿但消耗时间或钱，行政行动无聊但会打开学校注册/居留许可等关键门槛。金钱系统区分“可支配现金”和“冻结账户余额”：冻结账户不是一次性现金池，而是按月释放。

打工不再只是“换钱行动”。合法打工会累计 `current_week_work_hours` 和 `annual_work_half_days`，并按 2026 年德国法定最低工资 `13.90 EUR/小时` 自动结算收入：

- 普通打工约增加 10 小时，收入约 `139 EUR`，适合短期补现金。
- 多打一班工约增加 18 小时，收入约 `250 EUR`。如果同周已经打工，很容易超过 20 小时红线。
- 超过 20 小时会触发 `work_limit_exceeded_warning`，玩家可以减少排班、咨询许可、转 HiWi，或继续超时并承担 `work_law_violation` 风险。
- 资金为负时，每周漂移会额外增加压力和饥饿；在低钱且高饥饿状态下，会出现黑工诱惑事件。黑工按法定最低工资的 `80%` 现金结算，2026 年约 `11.12 EUR/小时`，低于最低工资，不计入合法工时，但会通过隐藏 flag 引出居留和法律风险。

恢复类行动使用冷却组和递减收益控制，不再允许同类恢复无限堆叠。`sleep_recover`、`bilibili_rest`、`therapy`、做饭和饭局都有每周上限或同组限制；最近几周重复使用同类恢复时，精力、压力、孤独和饥饿等恢复效果会打折。

## 情感支线

情感支线不是单纯的孤独值解药，而是不同对象类型带来的长期取舍：

- 稳定型对象：解锁 `partner_cook`，可以一起做饭、降低孤独和饥饿，并缓解考试压力。
- 消费型对象：解锁 `partner_spender`，短期大幅降低孤独、提升社交，但会触发高消费约会和现金流风险。
- 不稳定对象：解锁 `partner_unstable`，短期亲密感强，但会触发被欺骗感情、借钱、消失和创伤事件。
- 慢慢观察：解锁 `romance_slow`，收益较低，但风险也较低。

严重风险通过隐藏 flag 表达：`romance_financial_pressure` 会触发感情开销失控，`romance_scammed` 表示被欺骗感情，`romance_bankrupt` 会进入高优先级失败结局。

## 事件系统

事件分三类：

- fixed：固定周触发。
- conditional：条件满足触发。
- random：按权重随机触发。

事件可以有多个选项，选项支持条件、成功率、成功/失败效果、设置 flag 和后续事件。

当前实现要求每个事件在运行时都有 4 个选项。手写选项不足 4 个时，系统会按事件主题自动补齐：

- 稳妥处理：成功率中等，收益稳定，压力小幅增加。
- 寻求帮助：受社交和德语影响，成功后降低压力或提升关系，失败会增加压力。
- 冒险推进：基础成功率较低，成功收益高，失败损失精力并增加压力。
- 暂时回避：基础成功率较高，通常降低压力或恢复精力，但会牺牲事件核心进度。

成功率不是固定值，而是动态检定：

```text
最终成功率 = clamp(基础成功率 + Σ(当前属性值 * 属性系数), 5%, 90%)
```

例如行政/居留事件通常受德语和行政熟练度影响，课堂事件受学业和精力影响，社交事件受社交和德语影响，职业事件受职业进度和德语影响，生活事件会受精力、金钱或饥饿状态影响。压力、孤独等状态通常会降低成功率。

## 风险提示

主界面使用 `RiskEvaluator` 计算 top 3 风险。风险项包含分数、原因和建议行动，覆盖学校注册、现金流、学业/考试、压力、饥饿、居留、TestDaF 和打工合规。目标是让玩家每周知道当前最危险的三件事，而不是只看到一排静态属性。

## 难度模式

游戏支持四档难度：

- easy：轻松模式，给第一次体验的玩家更多容错。
- normal：标准模式，维持默认第一学期 Demo 节奏。
- hard：困难模式，行政、金钱和压力问题更容易滚雪球。
- realistic：现实模式，生活成本更高，恢复更慢，风险事件更频繁。

难度会影响三类规则：

- 每周生活漂移：不同难度下，精力恢复、压力增长、孤独增长、饥饿增长和生活费不同。
- 事件触发权重：固定事件仍按周触发；条件事件和随机事件会根据难度、事件类型和事件主题调整权重。
- 事件选项成功率：easy 提高需要检定的选项成功率，hard / realistic 降低需要检定的选项成功率，并调整成功率上下限。基础成功率为 100% 的确定动作不受难度影响。

事件 trigger 支持声明难度限制：

```text
difficulty: "hard"
difficulties: ["hard", "realistic"]
min_difficulty: "hard"
max_difficulty: "normal"
```

这样后续可以设计只在高难度出现的行政危机事件，或只在低难度出现的新手保护事件。

## 考试系统

考试由学业进度、备考掌握度、德语、精力、压力和随机波动共同决定，最后映射为德国成绩：1.0、1.3、1.7、2.0、2.3、2.7、3.0、3.3、3.7、4.0、5.0。

当前 Demo 公式：

```text
raw_score =
  academic_progress * 0.35
  + exam_readiness * 0.40
  + language * 0.10
  + (energy - 50) * 0.08
  - max(0, stress - 55) * 0.35
  + random(-8, 8)
```

考试未通过会增加 `failed_courses`，设置 `needs_retake`，额外增加压力并降低备考信心。成功类结局要求 `max_failed_courses = 0`，避免挂科后仍被判为稳定成功。

## 结局

Demo 结局包含稳定开局、学霸高压、社牛开局、职业起步、打工战神、行政崩盘、注册失败、考试失利、被迫离境、感情破产、心态爆炸、打工违规风险、延迟入学、现金流断裂、生活失衡、高压休整和勉强撑住。严重失败类和生存危机结局优先判定；成功类结局都有钱、饥饿、压力、注册、学业、备考和挂科底线，避免把现金流/学业严重失衡误报成成功。
