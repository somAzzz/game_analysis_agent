# Balance, economy, difficulty, and progression

Freeze the intended pressure curve before calling a result defective. Record
which personas may fail, acceptable scarcity, recovery windows, pacing, and
protected difficulty.

Review:

- inflow/outflow per interval and cumulative drift;
- mandatory versus optional costs;
- minimum spendable reserve and recovery actions;
- compounding stress/debt/health loops;
- action opportunity cost and dominant strategies;
- progression gates, time budgets, and irreversible traps;
- outcome distribution by persona, scenario, difficulty, and seed.

Prefer trajectory metrics: first crisis step, duration in crisis, recovery
probability, and area under resource pressure. A final-state average hides
when the design became unrecoverable.

When changing a numeric parameter, run a narrow sensitivity sweep first. Lock
units and update frequency; a value applied weekly is not comparable to one
applied once. Accept only when target cohorts improve, holdouts confirm, route
diversity/persona separation remain, and designed failure is still possible.
